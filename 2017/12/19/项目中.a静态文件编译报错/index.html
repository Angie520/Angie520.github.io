OC 项目中.a静态文件编译报错

2017.12.19 Angie OC  热度 ℃

.a静态库编译报链接错误，则很大原因是路径错了。查看路径是不是错了，是不是写成绝对路径了。

还有是其他的原因，比如是.a静态库不支持框架。

通过在终端输入 lipo -info xxx.a 来查看支持的架构。

结果是

说明这个.a静态库支持的是armv7 arm64，2个架构，如果不是这2个架构的就会编译失败。

armv7 :iPhone 4s

armv7s :iPhone 5

i386 :32位Mac os

x86_64 :64位Mac os

arm64 :iPhone 5s及其以上机型。

debug 模式下可能只会编译所属机型需要的架构，在project -> build Settings -> Build Active Architecture Onlu 可以设置是不是要只生成当前机型的静态库。debug无所谓，但release一定要设为NO，要生成所以支持的机型静态库。虽然会导致编译速度慢。
